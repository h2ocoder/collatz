## Accounts

### `BountyEscrow` (PDA)

Seeds: `["bounty", bounty_id: [u8; 32]]`  
Space: 400 bytes

| Field              | Type        | Description                                                   |
|--------------------|-------------|---------------------------------------------------------------|
| `bounty_id`        | `[u8; 32]`  | SHA-256 of off-chain bounty UUID                              |
| `buyer`            | `PublicKey` | Who posted the bounty                                         |
| `miner`            | `PublicKey` | Accepted miner (`Pubkey::default()` until accepted)           |
| `has_miner`        | `bool`      | True after a miner accepts                                    |
| `usdc_mint`        | `PublicKey` | USDC mint address                                             |
| `vault_ata`        | `PublicKey` | Vault ATA address (ATA of this PDA for `usdc_mint`)           |
| `predicate_hash`   | `[u8; 32]`  | SHA-256 of the predicate JSON constraint                      |
| `max_price`        | `u64`       | Maximum payout in USDC lamports (6 decimals)                  |
| `protocol_fee`     | `u64`       | 1% of `max_price`, retained by protocol                       |
| `bond_amount`      | `u64`       | 5% of `max_price`, staked by miner at acceptance              |
| `deadline`         | `i64`       | Unix timestamp — acceptance must occur before this            |
| `state`            | `BountyState` | State machine (see below)                                   |
| `deliverable_hash` | `[u8; 32]`  | SHA-256 of delivered git bundle (`[0;32]` until delivered)    |
| `has_deliverable`  | `bool`      | True after miner submits deliverable                          |
| `attestation`      | `[u8; 64]`  | Miner's Ed25519 signature over `deliverable_hash` (v1: stored only) |
| `delivered_at`     | `i64`       | Unix timestamp of delivery                                    |
| `bump`             | `u8`        | PDA bump seed                                                 |

### `BountyState` (enum)

| Variant     | Value | Meaning                                  |
|-------------|-------|------------------------------------------|
| `Open`      | 0     | Posted, awaiting miner acceptance        |
| `Accepted`  | 1     | Miner accepted and staked bond           |
| `Delivered` | 2     | Deliverable submitted; awaiting release  |
| `Released`  | 3     | Escrow released to miner                 |
| `Cancelled` | 4     | Buyer cancelled before acceptance        |

## Vault (ATA)

The USDC vault is the Associated Token Account (ATA) of the `BountyEscrow`
PDA for the given `usdc_mint`.

Derive via:
```ts
import { getAssociatedTokenAddress } from "@solana/spl-token";
const vaultAta = await getAssociatedTokenAddress(usdcMint, escrowPda, true);
```

The PDA owns the ATA and signs vault transfers via PDA seeds.

## Instructions

### `post_bounty`

Creates the escrow and deposits USDC.

**Accounts:**

| Name                     | Writable | Signer | Description                                  |
|--------------------------|----------|--------|----------------------------------------------|
| `buyer`                  | ✓        | ✓      | Pays rent + USDC deposit                     |
| `bounty_escrow`          | ✓        |        | PDA (created here)                           |
| `vault`                  | ✓        |        | Vault ATA (created here, owner = PDA)        |
| `usdc_mint`              |          |        | USDC mint                                    |
| `buyer_usdc`             | ✓        |        | Buyer's USDC ATA                             |
| `token_program`          |          |        | `TokenkegQfeZyiNwAJbNbGKPFXCWuBvf9Ss623VQ5DA` |
| `associated_token_program`|         |        | `ATokenGPvbdGVxr1b2hvZbsiqW5xWH25efTNsLJe8bv8Lh` |
| `system_program`         |          |        | `11111111111111111111111111111111`            |

**Parameters:**
```
bounty_id:      [u8; 32]   SHA-256 of bounty UUID
predicate_hash: [u8; 32]   SHA-256 of predicate JSON
max_price:      u64        USDC lamports
deadline:       i64        Unix timestamp
```

**Deposits:** `max_price + (max_price × 1%)` USDC from `buyer_usdc` → `vault`.

---

### `accept`

Miner stakes a bond and locks in.

**Accounts:**

| Name            | Writable | Signer | Description                         |
|-----------------|----------|--------|-------------------------------------|
| `miner`         | ✓        | ✓      | Miner accepting the bounty          |
| `bounty_escrow` | ✓        |        | Existing PDA                        |
| `vault`         | ✓        |        | Vault ATA (verified via stored key) |
| `usdc_mint`     |          |        | USDC mint (verified via stored key) |
| `miner_usdc`    | ✓        |        | Miner's USDC ATA                    |
| `token_program` |          |        |                                     |
| `system_program`|          |        |                                     |

**Transfers:** `bond_amount` (5% of `max_price`) from `miner_usdc` → `vault`.

**Requires:** `state == Open` and `now < deadline`.

---

### `deliver`

Submit a delivered universe.

**Accounts:**

| Name            | Writable | Signer | Description                              |
|-----------------|----------|--------|------------------------------------------|
| `miner`         |          | ✓      | Must match accepted miner in escrow      |
| `bounty_escrow` | ✓        |        | Existing PDA                             |

**Parameters:**
```
deliverable_hash: [u8; 32]   SHA-256 of git bundle
attestation:      [u8; 64]   Ed25519 sig over deliverable_hash (stored, not verified on-chain in v1)
```

**Requires:** `state == Accepted` and `miner == bounty_escrow.miner`.

---

### `verify_and_release`

Release escrow to the miner.

**Accounts:**

| Name            | Writable | Signer | Description                                                  |
|-----------------|----------|--------|--------------------------------------------------------------|
| `authority`     |          | ✓      | Buyer (immediate) or any key (after 24h auto-accept)         |
| `bounty_escrow` | ✓        |        | Existing PDA                                                 |
| `vault`         | ✓        |        | Vault ATA                                                    |
| `usdc_mint`     |          |        | USDC mint                                                    |
| `miner_usdc`    | ✓        |        | Miner's USDC ATA (owner verified against `escrow.miner`)     |
| `token_program` |          |        |                                                              |
| `system_program`|          |        |                                                              |

**Transfers:** `max_price + bond_amount` from `vault` → `miner_usdc`.  
`protocol_fee` remains in vault (swept by treasury separately).

**Requires:** `state == Delivered`, and either `authority == buyer` OR
`(now - delivered_at) >= 86400`.

---

### `cancel`

Cancel before any miner accepts.

**Accounts:**

| Name            | Writable | Signer | Description                                  |
|-----------------|----------|--------|----------------------------------------------|
| `buyer`         |          | ✓      | Must match `bounty_escrow.buyer`             |
| `bounty_escrow` | ✓        |        | Existing PDA                                 |
| `vault`         | ✓        |        | Vault ATA                                    |
| `buyer_usdc`    | ✓        |        | Buyer's USDC ATA (refund destination)        |
| `usdc_mint`     |          |        | USDC mint                                    |
| `token_program` |          |        |                                              |
| `system_program`|          |        |                                              |

**Transfers:** `max_price` from `vault` → `buyer_usdc`.  
`protocol_fee` retained as cancellation fee.

**Requires:** `state == Open`.

## Error codes

| Code              | Message                                                      |
|-------------------|--------------------------------------------------------------|
| `DeadlineInPast`  | Deadline must be in the future                               |
| `ZeroPrice`       | max_price must be greater than zero                          |
| `Overflow`        | Arithmetic overflow                                          |
| `InvalidState`    | Invalid state transition for this instruction                |
| `BountyExpired`   | Bounty deadline has already passed                           |
| `Unauthorized`    | Wrong signer or auto-accept window not yet elapsed           |
| `NoMiner`         | No miner assigned to this bounty                             |
| `InvalidMint`     | Provided mint or vault does not match escrow record          |

## v1 Verification Limitations

On-chain verification of universe mining is limited in v1:

- The `attestation` field (miner's Ed25519 signature over `deliverable_hash`) is
  **stored** in the escrow but **not verified on-chain**.  Ed25519 verification
  via Solana's `ed25519_program` syscall is planned for v2.
- Release currently requires buyer approval or a 24-hour auto-accept fallback.
- A future oracle network will re-run the universe mining from the VRF seed
  to cryptographically verify predicate satisfaction on-chain.

import { getAssociatedTokenAddress } from "@solana/spl-token";
import * as crypto from "crypto";

anchor.setProvider(provider);
const program = anchor.workspace.BountyEscrow;

// Compute bounty_id from UUID
